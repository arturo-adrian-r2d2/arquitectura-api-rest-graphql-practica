# Arquitectura de Comunicación

## Descripción general
La arquitectura propuesta sigue un modelo cliente-servidor basado en APIs
para permitir la interconectividad entre sistemas heterogéneos.

## Flujo de comunicación
Cliente → API → Servidor → Base de Datos → Respuesta al Cliente

## Justificación
Una arquitectura de comunicación eficiente reduce la latencia, el consumo de datos
y mejora la experiencia del usuario final, independientemente del rendimiento de la
base de datos.
